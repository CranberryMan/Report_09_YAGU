# Report_09_YAGU
the game of YAGU

https://youtu.be/BtmsdhRREys?si=xoTvLFD8H6cLJnxp

<h1>필수 기능 게임</h1>
1. 게임 모드 베이스의 판정 로직 구현<br>
 - 멀티 플레이 채팅 구현<br>
 - 정답 숫자 랜덤 생성<br>
 - 채팅 내용이 답안 제출인지 확인하는 기능 구현<br>
 - 답안을 판정하여 S/B/Out을 확인하는 기능 구현<br><br>
2. PlayerState를 활용한 시도 횟수 관리<br>
 - 각 플레이어 최대 횟수 3회 관리<br>
 - 턴 진행중 1회에 한해 답안 제출 시도를 할 수 있고, 시도시 횟수가 오르도록 구현<br>
3. 승리, 무승부, 리셋<br>
 - 게임 결과를 공지 위젯을 통해 알림<br>
 - 게임 결과가 나오면 5초 뒤 게임을 리셋 후 재시작<br><br><br>

<h1>도전 기능</h1>
1. 연속 채팅<br>
 - 채팅 입력시마다 Edit Text box를 클릭해야 하는게 불편해서, 채팅 입력 후 자동으로 다시 Edit Text box에 포커스를 이동시켜 연속으로 채팅을 칠 수 있게 하였습니다.<br><br>
2. 실시간 턴제<br>
 - 숫자 야구라는 게임 자체가, 너 한 번, 나 한 번 하는 방식으로는 불합리 하다고 생각했습니다.<br>
   * 2번 턴에서 플레이어1이 먼저 답을 입력했을 경우, 플레이어2는 정답 제출의 기회를 박탈당하고 패배할 수밖에 없는 불합리함<br>
 - 실시간 턴제 방식을 이용해, 턴이 시작되면 모든 플레이어가 동시에 시간이 차감됩니다. 해당 턴 안에 "먼저" 정답을 제출한 사람이 승리합니다.<br>
 - 타 플레이어의 오답을 보고 S/B/Out 여부를 확인한 뒤 자신의 답을 작성하는 등 전략적인 플레이가 가능합니다.<br>
 - 한 턴 안에 여러번 정답 제출을 시도할 시 "Your turn is off"라는 알림 메시지가 출력되며, 정답 제출로 인정되지 않습니다.<br><br>
3. 자동 시작<br>
 - 플레이어가 한 명만 있을 때는 게임이 시작되지 않으며, 플레이어2가 접속하면 잠시 후 게임을 시작합니다.<br>
 - 플레이어1과 플레이어2가 게임을 진행하는 중에 플레이어3이 접속할 경우 플레이어3은 해당 게임이 종료된 후 다음 게임이 시작할 때 게임에 참여합니다.<br>
 - 플레이어1과 플레이어2가 게임을 진행하는 중, 플레이어3은 플레이어1, 2와 채팅을 통해 대화할 수 있지만 숫자 야구 기능은 동작하지 않습니다.(정답 제출, 정답 판정, 턴관련 기능 등)<br><br>
4. 채팅 기록<br>
 - 채팅 내역을 Log가 아닌 Widget에 남겨 채팅이 사라지지 않고 보여집니다.<br>
    * 채팅 내역은 현재 10줄까지만 기록하고 그 이상의 기록은 제거하고 있습니다.<br>
